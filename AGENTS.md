# Project Instructions

## General

Use project conventions before introducing new patterns.

Before editing:

- inspect existing files;
- understand the framework;
- identify build, test and lint commands.

## Frontend

For frontend work, use the frontend-related skills:

- frontend
- ui-review
- accessibility
- refactor-component
- rails-views

Frontend priorities:

1. Clean and responsive UI.
2. Reusable components.
3. Accessibility basics.
4. Consistent spacing, colors and typography.
5. No unnecessary dependencies.
6. Preserve existing behavior.

## Rails Views

When working in Ruby on Rails views:

- prefer partials to avoid duplication;
- keep business logic out of ERB when possible;
- reuse existing helpers, layouts and conventions;
- prefer Rails path helpers;
- keep HTML semantic and accessible;
- avoid adding gems unless truly necessary.

## Verification

After changes, run the most relevant available command:

- tests;
- lint;
- typecheck;
- build.

If no command exists, explain how to test manually.
